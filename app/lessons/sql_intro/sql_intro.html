SQL Introduction

What is SQL?

Structured Query Language, or SQL is a domain-specific language language for manipulating data stored in a SQL database.

SQL was invented in 1974 based off of a paper by Edgar F. Codd, A relational model of data for large shared data banks.

In SQL, data is stored in tables, and accessed with regular queries.

SQL tables consist of intersecting rows and columns that form a 2 dimensional grid with data held in individual cells within this grid.

Each Column and Row has a label that can be used to query it

SQL is used on relational databases, and this means that tables can reference other tables. Thanks to this feature, data can be stored in a compact and reusable manner.

Who uses SQL?

SQL has been adopted as a standard by the American National Standards Institute, the ANSI, in 1986 and by the International Organization for Standardization, the ISO, in 1987.

The language has since grown to be used in millions of applications both on and off the web.

36% of the participants in a Poll on StackOverflow confirmed that they have recently worked with SQL

In 2015, Evans Data Corporation found that there are approximately 19 million developers who use SQL in the world

Why learn SQL?

Today, SQL has implementations in nearly every popular programming language, and thanks to its easy syntax, it is a great database for beginners who haven't worked with many database systems

SQL is a standardized Specification, which means that the implementation may vary depending on how the designers of the implementation interpret the specification.

SQL has been baked into many large public and proprietary systems and as such there will always be a demand for SQL developers to maintain these systems

SQL Statements are really powerful, but many web applications use form data and other forms of insecure data to construct SQL statements. As problems from this anti-pattern pop up, the market will need cyber security experts who are familiar with and can recognize SQL exploits

SQL: A Brief Crash Course

SQL Syntax

SQL operations are Statements that declare what you want from the database and how to change it

An SQL Statement consists of a command followed by other commands, keywords, or names

For example the following SQL statement will create a new table inside the database called 'temp'

CREATE TABLE temp

The SELECT command is the most common way to query data from a SQL table, with wild-cards and logic to refine the search parameters. For Example:

SELECT * from temp

The above command would return every row and column from the temp Table

Data Types

Integer: Stores positive and negative whole numbers

Float: Stores numbers that contain a decimal point

Datetime: Stores the time and data in the format YYY-MM-DD HH:MM:SS

Varchar: Stores a limited number of single characters

Blob: Stores Binary Data

Important Keywords

ADD: Adds a column in an existing table

DELETE: Deletes data from a database

DROP: Similar to DELETE

FROM: Specifies which table to select or delete data from

JOIN: Joins tables

SELECT: Extracts data from a database

UPDATE: Updates data in a database

Common Keywords

ALL: Returns true if all the subcondition's values are true

AND: Only include rows where both conditions are true

ANY: Returns true if any of the subcondition's values are true

AS: Renames a column or table with an alias

IN: Allows you to specify multiple values in a WHERE clause

NOT: Only includes rows where a condition is not true

OR: Includes rows where either condition is true

WHERE: Filters a result set to include only records that fulfill a specified condition

Other Keywords

INSERT INTO: Insert new data into a database

CREATE DATABASE: Create a new database

ALTER DATABASE: Modify a database

CREATE TABLE: Create a new table

ALTER TABLE: Modify a table

DROP TABLE: Delete a table

CREATE INDEX: Create an index

DROP INDEX: Delete an index
